# 인프라 시작 — jaringobe

## 사용 예시
```
# DB 스키마 변경 (구현 에이전트 요청 처리)
/인프라시작 fridge_item 테이블에 expires_at 컬럼 + 인덱스 추가

# 마이그레이션만
/인프라시작 [마이그레이션] 최신 모델 기준 Alembic revision 생성

# Docker/로컬 환경
/인프라시작 [환경] docker-compose 에 postgres 16 + 초기 스키마 구성

# 초기 프로젝트 스캐폴딩
/인프라시작 [스캐폴딩] frontend/backend 모노레포 초기 구조 생성
```

---

아래 순서로 진행한다.

## 1. 에이전트 하네스 로드
`agents/infra.md` 파일을 읽고 인프라 에이전트 규칙을 완전히 숙지한다.

## 2. 입력 확인
```
$ARGUMENTS
```

## 3. 작업 유형 분기

### DB 스키마 변경 (GATE 3)
```
1. 기존 스키마 분석 (backend/app/models/ + alembic history)
2. 영향도 분석 (FK/인덱스/기존 서비스 영향, 영향 도메인 목록)
3. 마이그레이션 계획 보고 → "DB 승인" 대기
4. Alembic 스크립트 작성 (upgrade + downgrade 필수)
5. 로컬 DB에서 upgrade → downgrade → upgrade 검증
6. docs/설계/db-schema.md 갱신 + 영향 에이전트 통보
```

### 환경/배포 설정
```
1. 현재 설정 분석 (docker/, docker-compose.yml, .env.example, CI)
2. 변경 계획 보고 → 승인
3. 적용 + 로컬 검증 (docker compose up 확인)
```

## 4. jaringobe 특화 검증 (반드시)
- 금액 컬럼 `numeric` + 통화 코드 / 시각 컬럼 `timestamptz` (UTC)
- 마트 자격증명 컬럼 평문 저장 스키마 여부
- 누적성 테이블(식단 이력/냉장고 로그/주문 이력) 인덱스 전략
- `.env` 커밋 여부 / 시크릿 노출 여부
