# 기획 시작 — jaringobe

## 사용 예시
```
# 직접 요구사항
/기획시작 가상 냉장고 유통기한 임박 알림 기능. 푸시 없이 인앱 배너 우선

# 추가 제약과 함께
/기획시작 자동 주문 주기 설정 기능 / 추가: 미국(Instacart)도 동일 UX 로 동작해야 함

# 도메인 지정
/기획시작 [budget] 예산 락 초과 시 식단 재조정 흐름
```

---

아래 순서로 진행한다.

## 1. 에이전트 하네스 로드
`agents/planning.md` 파일을 읽고 기획 에이전트 규칙을 완전히 숙지한다.

## 2. 제품 방향 확인
`docs/사업/사업계획서-요약.md` 를 읽고 요구사항이 3대 축(예산 락 식단 / 가상 냉장고 / 자동 주문)과 선순환 루프에 어떻게 기여하는지 확인한다.

## 3. 입력 정보 통합
```
$ARGUMENTS
```

통합 기준:
- 요구사항 본문 → 공식 요구사항 및 배경
- `추가:` 뒤 내용 → 기술 방향, 우선순위, 제약사항, 제외 항목
- `[도메인]` 태그 → auth/household/budget/mealplan/fridge/order/store/subscription 중 중심 도메인

## 4. 기획 에이전트 시작
`agents/planning.md` 규칙에 따라 6개 서브 에이전트 토론을 시작한다.

- 토론 전 통합된 입력 정보를 PM 에이전트에게 전달
- PM → Devil's Advocate → UX → 기술 → 보안 → 데이터 순으로 토론 진행
- **영향 영역을 반드시 식별** (frontend/backend/DB/마트 연동)
- **글로벌 고려사항 명시** (KR/US 동작 차이, i18n, 통화)
- 합의 완료 후 `docs/기획/{기능명}.md` 저장

## 5. 완료 보고
```
## 기획 완료 보고
- 산출물: docs/기획/{기능명}.md
- 영향 영역: {frontend/backend/DB/마트 연동}
- 관련 도메인: {목록}
- 글로벌 고려: {요약}
- 주요 결정사항: {요약}
- 미결 사항: {있을 경우}
- GATE 1 승인 요청: "기획 승인" 또는 "설계 시작해" 명령을 입력해주세요
```
