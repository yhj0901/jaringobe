# 문서 시작 — jaringobe

## 사용 예시
```
# 기본 (QA 완료 후 자동 실행)
/문서시작

# 릴리즈 노트만
/문서시작 [릴리즈] v0.2.0 릴리즈 노트만 작성

# 특정 문서만
/문서시작 [기술문서] store-adapter-guide.md 만 갱신
```

> 이 커맨드는 QA 완료(Critical/High 0건) 후 자동 실행되거나 오케스트레이터가 직접 호출할 수 있다.

---

아래 순서로 진행한다.

## 1. 에이전트 하네스 로드
`agents/docs.md` 파일을 읽고 문서 에이전트 규칙을 완전히 숙지한다.

## 2. 입력 확인
```
$ARGUMENTS
```
입력이 없는 경우 전체 산출물 자동 로드:
- `docs/사업/` `docs/기획/` `docs/설계/` `docs/테스트/` + 구현 코드 diff

## 3. 문서 작성
- 기술 문서 + 운영 가이드 병렬 작성 (`docs/기술문서/`, `docs/운영가이드/`)
- 릴리즈 노트 작성 (`docs/릴리즈노트/v{버전}.md`)
- **산출물 기반으로만 작성 — 임의 창작 금지**

## 4. 문서 검토관 최종 검토
누락/오류/일관성 검토 → 지적사항 반영

## 5. 전체 파이프라인 완료 보고
```
## 전체 파이프라인 완료 보고
### 산출물 목록
### 영향 영역 요약 (frontend/backend/DB/마트 연동)
### DB 마이그레이션 필요 여부
### 주요 결정사항 요약
### 알려진 이슈 (Medium/Low 버그)
### 다음 버전 제안사항
```
