# 설계 시작 — jaringobe

## 사용 예시
```
# 기본 (최신 기획문서 자동 로드)
/설계시작

# 특정 기획문서 지정
/설계시작 docs/기획/가상냉장고-임박알림.md

# 설계 범위 제한
/설계시작 API 스펙과 DB 스키마만 먼저 설계
```

> 이 커맨드는 오케스트레이터의 GATE 1 승인 후에만 실행한다.

---

아래 순서로 진행한다.

## 1. 에이전트 하네스 로드
`agents/design.md` 파일을 읽고 설계 에이전트 규칙을 완전히 숙지한다.

## 2. 입력 확인
```
$ARGUMENTS
```
입력이 없는 경우 자동으로 로드:
- `docs/기획/` 의 최신 기획문서
- `docs/사업/사업계획서-요약.md`
- 기존 `docs/설계/` 산출물 (하위 호환성 검토용)

## 3. 설계 토론 시작
`agents/design.md` 규칙에 따라 5개 서브 에이전트 토론을 진행한다.

- 시스템 아키텍트 → API 설계자 + DB 설계자 → UI 대변인 → 보안 검토관 → 조율
- 금액(Decimal+currency)/시간(UTC)/i18n/마트 어댑터 원칙 준수 (글로벌 전제)
- API 스펙은 UI 대변인 동의 후 확정

## 4. 산출물 저장 (`docs/설계/`)
- architecture.md / api-spec.md / db-schema.md / security-design.md
- store-adapter.md (마트 연동 변경 시) / ui-design.md (해당 시)

## 5. 완료 보고
```
## 설계 완료 보고
- 산출물: docs/설계/ {파일 목록}
- API 엔드포인트: {N}개
- DB 변경: {있음(테이블 목록)/없음}
- 마트 어댑터 변경: {있음/없음}
- 주요 결정사항: {요약}
- 미결 사항: {있을 경우}
- GATE 2 승인 요청: "설계 승인" 명령을 입력해주세요
  (승인 후 /API시작 + /UI시작 병렬 진행, DB 변경 있으면 /인프라시작 선행)
```
