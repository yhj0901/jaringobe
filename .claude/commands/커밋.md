# 커밋 — jaringobe

## 사용 예시
```
# 메시지 자동 생성 (변경 내용 기반)
/커밋

# 메시지 직접 지정
/커밋 feat: 가상 냉장고 자동 차감 API 구현

# 브랜치 지정 커밋
/커밋 feature/fridge-auto-deduct fix: 재고 차감 경쟁 조건 수정
```

> 브랜치 확인(필요 시 생성) → 커밋 → 푸시를 진행한다. **모든 커밋 전 사용자 승인 필수.**

---

아래 순서로 진행한다.

## 1. 입력 파싱
```
$ARGUMENTS
```
- 커밋 메시지 (선택 — 없으면 변경 내용 기반으로 자동 생성)
- 브랜치명 (선택)

## 2. 변경 확인
```bash
git status --short
git diff --stat
```
- 변경 파일 목록 + 변경 행 수 확인
- `.env`, 시크릿, 대용량 바이너리가 포함되어 있으면 즉시 사용자에게 보고하고 중단

## 3. 브랜치 확인
- 현재 브랜치가 `main` 이면 → 커밋 금지. `feature/{기능}` / `fix/{버그}` 브랜치 생성 여부를 사용자에게 확인
- 이미 feature/fix 브랜치이면 → 그대로 진행

## 4. 커밋 메시지 규칙
```
형식  : {type}: {내용}   (한국어)
type  : feat / fix / refactor / style / test / docs / chore / db
예시  : feat: 예산 락 초과 시 식단 재조정 API 추가
        db: fridge_item 테이블 expires_at 인덱스 추가
```
- DB 마이그레이션 포함 커밋은 반드시 `db:` prefix
- i18n 리소스만 변경 시 `feat:` 또는 `fix:` + 내용에 "(i18n)" 표기

## 5. 승인 요청

**반드시 사용자 승인을 받는다**:

```
## 커밋 요약
- 브랜치: {브랜치명}
- 변경 파일:
  {파일 목록 + 변경 행 수}
- 커밋 메시지: {type}: {메시지}

커밋 + 푸시 진행할까요?
```

## 6. 커밋 + 푸시 (승인 후)
```bash
git add {변경 파일들}
git commit -m "{type}: {메시지}"
git push -u origin {브랜치명}
```

## 7. 완료 보고
```
## 커밋 완료
- 커밋: {short hash}
- 브랜치: {브랜치명}
- 푸시: ✅
```

---

## 주의사항
- `main` 브랜치 직접 커밋 금지
- `.env` / 시크릿 / 마트 자격증명 포함 파일 커밋 금지
- 테스트 미통과(커버리지 90% 미만) 상태 커밋 금지 — 예외는 사용자 명시 승인 시에만
- 모든 커밋 전 **사용자 승인 필수**
