# QA 시작 — jaringobe

## 사용 예시
```
# 기본 (자동 실행 - 모든 구현 에이전트 완료 후)
/QA시작

# 특정 기능만 QA
/QA시작 가상 냉장고 자동 차감 기능만 테스트

# 버그 수정 후 회귀 테스트
/QA시작 [회귀] BUG-012 수정 후 재테스트

# 테스트 서버 정보 포함
/QA시작 서버: http://localhost:3000 / API: http://localhost:8000

# 보안 테스트만 집중
/QA시작 [보안] 인증/인가 + 자동 주문 예산 상한 우회 집중 검증

# 도메인 루프 테스트
/QA시작 [루프] 식단→주문→냉장고→감산 선순환 루프 정합성 검증

# i18n 테스트
/QA시작 [i18n] ko/en 키 누락 + 통화/날짜 포맷 검증
```

> 이 커맨드는 모든 구현 에이전트 완료 후 자동 실행되거나 오케스트레이터가 직접 호출할 수 있다.

---

아래 순서로 진행한다.

## 1. 에이전트 하네스 로드
`agents/qa.md` 파일을 읽고 QA 에이전트 규칙을 완전히 숙지한다.

## 2. 입력 확인
```
$ARGUMENTS
```
입력이 없는 경우 전체 산출물 자동 로드.

## 3. 테스트 계획 수립
```
## 테스트 계획
- 유저 스토리 커버리지: {N}/{N}
- API 엔드포인트 커버리지: {N}/{N}
- 도메인 루프 테스트: {N}건
- 보안 테스트 케이스: {N}건
- i18n 테스트 케이스: {N}건
- 총 테스트 케이스: {N}건
```

## 4. API 통합 + 도메인 루프 + 보안 + i18n 테스트 (병렬)

## 5. 시나리오 테스트 (서버 정보 수신 후)
- 자동 주문/결제 관련은 반드시 테스트 계정/샌드박스만 사용

## 6. 버그 리포트 (`docs/테스트/버그리포트.md`)
```
BUG-001:
  심각도 / 담당([API]/[UI]/[인프라]/[설계])
  재현경로 / 기대결과 / 실제결과
```
**Critical/High 즉시 에스컬레이션**

## 7. 문서 자동 인계
Critical/High 0건 확인 후 → `/문서시작` 자동 실행
