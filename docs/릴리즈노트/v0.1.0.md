# v0.1.0 릴리즈 노트

- 릴리즈 날짜: 2026-07-09 / 이전 버전: 없음 (최초 기능 릴리즈)
- 범위: 게스트 홈 진입 경험 + 소셜 로그인 (브랜치 `feature/auth-guest-home`, QA GATE 4 통과)

## ⚠️ Breaking Changes
- **DB 마이그레이션 필요**: `alembic upgrade head` — 리비전 `0001_initial_auth_budget` (users/auth_identities/refresh_tokens/budget_plans 신규, 최초 리비전)
- 신규 필수 환경 변수: `JWT_SECRET` (없으면 인증 불가). 소셜 로그인엔 `KAKAO_*`/`GOOGLE_*` 키 필요. 배포 시 `FRONTEND_ORIGIN`·`COOKIE_SECURE=true`
- API v1 공통 규격 최초 확정: camelCase / `{"detail":{"code","message"}}` / 금액 문자열+통화

## 🔒 보안
- OAuth state 서명 토큰(CSRF), next 상대경로 화이트리스트(open redirect), refresh 회전+재사용 감지 전 세션 폐기, httpOnly 쿠키, Origin 검증·rate limit 미들웨어 — 보안 테스트 10/10 PASS (`docs/테스트/보안테스트.md`)
- 스캐폴딩의 CORS 전체 허용 제거 (rewrites 동일 오리진 구조로 대체)

## ✨ 신규 기능
- **게스트 홈** (US-101~104, 107): 샘플 데이터 홈 셸("체험 모드" 배지), 체류 10초+스크롤 유휴 시 예산안 제안 바텀시트(거절 시 세션 내 재노출 없음), 예산안 3스텝 작성 → 홈 전체 갱신(정적 샘플 매트릭스, 서버 호출 없음)
- **가입 전환** (US-105~106): 예산 적용 후 자동주문 제안 → 소셜 로그인 → 게스트 예산안 계정 이전(온보딩 스킵), localStorage 30일 복원
- **소셜 로그인** (US-001~006): 카카오·구글 OAuth + JWT 세션, provider별 별도 계정 정책, 로캘별 버튼 순서, 실패/거부 안내 배너. 애플은 P1 (인터페이스 확정, 404 응답)
- API 6종: authorize / callback / refresh / logout / users·me / budget·plans
