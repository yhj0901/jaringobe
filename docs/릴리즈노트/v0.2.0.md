# v0.2.0 릴리즈 노트

- 릴리즈 날짜: 2026-07-16 / 이전 버전: v0.1.0
- 범위: **앱 웹뷰 래핑 + 푸시 알림** (기획 `docs/기획/앱-웹뷰-푸시알림.md`, QA GATE 4 통과 — 버그 7건 수정 완료)

## ⚠️ Breaking Changes
- **API 계약 변경 (v1.5/v1.5.1)**: `POST /api/v1/mealplans`·`/{id}/regenerate` 가 동기 `201` → **`202 Accepted` 비동기**로 전환. 클라이언트는 `GET /mealplans/{id}` 폴링 필수. `MealPlanResponse.status` 가 `processing|ready|over_budget|failed` 4종으로 확장 (processing/failed 시 `meals: []`·`budgetSummary: null`·기간 null). 신규 에러 코드: `409 MEALPLAN_GENERATING`, `409 MEALPLAN_REGENERATE_EMPTY`
- **DB 마이그레이션 필요**: `alembic upgrade head` — 리비전 `0008_notification_app` (device_tokens / notification_settings / notification_logs / app_login_codes 신규, 전부 additive). **선행 조건: `feature/global-region` 의 리비전 0007 이 먼저 main 머지·적용**되어야 함 (down_revision=0007)
- 신규 환경 변수: `EXPO_ACCESS_TOKEN`(푸시 발송), `APP_SCHEME`(기본 jaringobe). 선택: `MEALPLAN_GENERATION_TIMEOUT_MINUTES`(기본 10) / `REMINDER_SCHEDULER_ENABLED`(기본 true) / `REMINDER_SCHEDULER_INTERVAL_SECONDS`(기본 30)

## 🔒 보안
- 앱 로그인 원타임 코드: SHA-256 해시만 저장·60초 만료·**원자적 단일 사용**(BUG-004), 실패 응답 무구분(oracle 차단), 재사용 시도 감사 로그
- 웹뷰 오리진 **완전 일치** 검증(BUG-003, CWE-345/601) + 외부 링크 시스템 브라우저 위임, 브리지 postMessage 출처 검증
- 푸시 본문 개인정보 정책: 메뉴명까지만(CWE-359), 발송 이력은 template_key 만 저장, 딥링크 path 상대경로 화이트리스트
- 보안 테스트 12항목 중 FAIL 3건 전부 수정 후 회귀 PASS (`docs/테스트/보안테스트.md`)

## ✨ 신규 기능
- **모바일 앱 쉘** (`mobile/` 신규, US-005): Expo + react-native-webview — 배포 웹을 웹뷰로 로드, 네이티브 스플래시·오프라인 재시도·Android 뒤로가기. 소셜 로그인은 3사 공통 커스텀 탭 + 원타임 코드 세션 인계(`GET /auth/app/session`)
- **식단 생성 완료 푸시** (US-001, FR-005): 생성 비동기화(BackgroundTasks) + 완료/실패 시 Expo 푸시. 웹/앱 공통 화면 폴링(3초→5초 백오프, 3분 한도) — 푸시는 보조 채널
- **식사 시간 리마인더** (US-002/003, FR-006): 아침/점심/저녁 개별 on/off·시각 설정, 사용자 타임존 기준(`next_send_at` UTC 인덱스 스캔), **당일 해당 끼니가 존재하고 미완료일 때만** 메뉴명 포함 발송
- **알림 설정 화면** (`/settings/notifications`, US-003): 행 단위 즉시 저장(낙관적 갱신), OS 권한 거부 배너, 웹 브라우저 안내 카드
- **웹↔앱 브리지 v1** (FR-004): BRIDGE_READY/PERMISSION_STATUS/PUSH_TOKEN ↔ REQUEST_PUSH_PERMISSION/OPEN_OS_SETTINGS/LOGIN_PROVIDER/SYNC_REQUEST — 미지 type 무시(전방 호환)
- **푸시 권한 soft ask** (FR-002): 첫 식단 생성 직후 1회 안내 후 OS 권한 요청
- API 신규 5종: notifications devices PUT/DELETE·settings GET/PUT + auth app/session (`docs/설계/api-spec.md` v1.5.1)

## 🐛 버그 수정 (QA 2026-07-14, BUG ID 기반)
- BUG-001 [High] 좀비 processing 플랜의 생성 영구 409 잠금 → 10분 타임아웃 수렴
- BUG-002 [High] failed 플랜 재생성 시 하루 끼니 수 1 로 붕괴 → `MEALPLAN_REGENERATE_EMPTY` + 프론트 생성 시트 전환
- BUG-003 [High] 웹뷰 오리진 프리픽스 매칭 우회 → origin 완전 일치
- BUG-004 [Medium] 원타임 코드 이중 소진 race → 원자 UPDATE
- BUG-005 [Medium] 다중 플랜 공존 시 구플랜 끼니 리마인더 오발송 → 최신 플랜 선선택
- BUG-006 [Medium] 초기 브리지 메시지 유실 → SYNC_REQUEST 프로토콜 증보
- BUG-007 [Low] 409 합류 시 완료 플랜을 실패로 표시 → completed 처리
