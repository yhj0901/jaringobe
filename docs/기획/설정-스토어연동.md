# 설정 페이지 + 자동 주문 스토어 연동 기획서

## 개요
프로토타입 설정 화면(스크린샷 확정, 2026-07-10)을 구현한다: **계정 / 내 식생활 설정(수정→식단 재생성) / 자동 주문 연동 스토어**.

## 범위
### In Scope
- `/settings` 신설 (홈 헤더 GB 아바타 → 진입, 뒤로가기)
- **계정**: 프로필(닉네임·이메일·로그인됨 배지) + **로그아웃**(기존 POST /auth/logout 연결, visited 마커 기록 — 원설계 회복)
- **내 식생활 설정** 3항목 — 현재값 요약 표시(N인 가구 / 방향·선호 / 월 예산), 클릭 시 **온보딩 스텝 컴포넌트를 단일 편집 모드로 재사용**, 저장은 기존 `PUT /households/me`·`PUT /budget/plans`
- **수정 저장 성공 시 "식단을 다시 만들까요?" 확인 시트** → 수락 시 재생성(기존 regenerate/생성 흐름 재사용), 거절 시 홈에서 나중에 가능
- **자동 주문 연동 스토어**: 컬리/쿠팡/SSG/네이버(KR — 브랜드 배지·색), 연동하기/해제 토글 — **1단계: 연동 상태 관리만** (자격증명 미수집, 연동 확인 시트 → 상태 저장). 연동됨 항목은 서비스 계정 이메일 표시
- 백엔드: `GET/PUT /api/v1/stores/connections` + **`store_connections` 테이블 신규(GATE 3)**

### Out of Scope (팀원 store/order 본기획 — 조율 항목)
- 마트 **실계정 자격증명 연동**(로그인 자동화·암호화 저장·약관 리스크 검토), 실제 자동 결제·주문 실행, US 스토어(Walmart/Instacart) 연동 UI
- 닉네임/프로필 수정, 회원 탈퇴(별도 기획)

## 영향 영역
frontend(설정 페이지·편집 재사용·재생성 확인) / backend(stores 연동 상태 API — store 도메인, 팀원 리뷰 필수) / **DB: store_connections 신규 → GATE 3** / 마트 연동: 상태 관리까지(실연동 없음)

## 관련 도메인
store(연동 상태 — 중심, 팀원 조율) · household/budget/mealplan(설정 수정·재생성) · auth(계정·로그아웃)

## 필요 API (초안 — 설계 확정)
- `GET /api/v1/stores/connections` → [{store, status, connectedAt}]
- `PUT /api/v1/stores/connections/{store}` body {connected: bool} → 200
- 기존 재사용: households/me, budget/plans(PUT), mealplans regenerate/생성, auth/logout, users/me

## 데이터 요구사항 (GATE 3)
- `store_connections`: user_id FK, store varchar CHECK('kurly','coupang','ssg','naver'), status('connected','disconnected'), connected_at, UNIQUE(user_id, store) — **자격증명 컬럼 없음**(실연동 시 팀원 설계로 암호화 참조 확장)

## 보안 (CWE)
CWE-639 본인 스코프 / CWE-20 store enum 검증 / 자격증명 미수집(신규 민감 표면 없음) / 로그아웃 시 세션 폐기 기존 로직

## 글로벌 고려
스토어 목록 로캘별(KR 4종 / US Walmart·Instacart 는 후속), 라벨 i18n ko/en

## 미결 사항
- 없음 (0건) — 실연동 방식·약관 리스크는 store 본기획(팀원)으로 명시 이관
