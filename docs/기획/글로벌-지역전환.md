# 글로벌(지역 전환) + 국가별 스토어 기획서

## 개요
한국 외(미국) 사용자를 위한 **수동 지역 전환**과 **국가별 자동 주문 스토어 교체**. 통화(KRW/USD)·예산 프리셋·샘플 식단(en 미국 가정식)은 이미 구축돼 있어, 이번엔 ① 지역(국가) 선택·저장·표시 ② 스토어 목록을 국가별로(KR 4종 ↔ US Walmart·Instacart) 전환한다.

- 감지 방식: **수동 선택**(설정에서 지역 전환) — IP/GPS 자동 감지는 이번 범위 아님(후속). 사용자 확정
- store 도메인·DB 제약 변경은 팀원 영역 → **리뷰 필수**, 기존 팀원 store 파일 무접촉(우리가 만든 connection_* + 마이그레이션만)

## 범위
### In Scope
- users.country(KR|US)·currency(KRW|USD) 를 사용자가 전환하는 지역 설정 (설정 페이지 "지역·통화" 행)
- country != KR 시 홈 헤더/설정에 "글로벌" 배지 표시
- 스토어 목록 국가별: KR → 마켓컬리·쿠팡·SSG·네이버 / US → Walmart·Instacart. GET/PUT /stores/connections 가 country 기준으로 동작
- store_connections CHECK 제약에 walmart·instacart 추가 (마이그레이션 0007) → GATE 3
- 지역 전환 시 통화·스토어·배지 즉시 반영. 신규 예산안/온보딩 통화 기본값도 country 따름

### Out of Scope (이관/후속)
- IP·GPS 자동 감지(후속), 기존 예산안/식단의 통화 소급 변환(전환은 신규 데이터·표시에만 — 기존 플랜은 저장 통화 유지), Walmart/Instacart 실 API 연동(팀원 store 본기획), 다국가 확장(현재 KR/US 2국)

## 영향 영역
frontend(지역 토글·글로벌 배지·국가별 스토어 리스트) / backend(users 지역 업데이트 API + stores/connections 국가별) / **DB: store_connections enum 확장 → GATE 3** / 마트 연동: 목록·상태까지(실 API 없음)

## 필요 API (초안)
- `PUT /api/v1/users/me/region` {country: "KR"|"US"} → currency 자동 매핑, 200 UserMe(country/currency 갱신)
- `GET /api/v1/stores/connections` — user.country 기준 스토어 세트 반환(기존 확장)
- `PUT /api/v1/stores/connections/{store}` — country 허용 스토어 검증(기존 확장)

## 데이터 요구사항 (GATE 3)
- 마이그레이션 0007: store_connections `ck_store_connections_store` 제약을 kurly/coupang/ssg/naver + **walmart/instacart** 로 재정의 (기존 행 영향 없음)
- users.country/currency 는 기존 컬럼 재사용 (스키마 변경 없음)
