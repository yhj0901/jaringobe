# 식사 완료 체크 + 레시피 시트 기획서

## 개요
주간 식단(1주 단위 생성·주문 방향)의 소비 접점 완성: **끼니별 완료 버튼**(프로토타입 UI)과 **끼니 클릭 → 레시피 바텀시트**(조리 방법). 선순환 루프 ③(식사 완료 시 자동 차감)의 전 단계 — 완료 상태 저장까지 하고, 냉장고 자동 차감은 팀원 fridge 본기획과 조율(이관).

## 범위
### In Scope
- 끼니 행: 완료 버튼(미완료=파란 CTA / 완료=연한 배지, 재터치로 해제 가능), 비용 표시 유지
- 레시피 바텀시트: 끼니 배지 + "AI 추천 레시피" 배지, 요리명, 메타 3칩(조리 시간·난이도·N인분=가구 인원), 재료 칩, 조리 방법 단계(1,2,3…), 닫기
- 백엔드: meals 완료 API(설정/해제) + MealOut 에 steps·completedAt·(옵션) timeMinutes·difficulty 노출, LLM 프롬프트에 time/difficulty 옵션 필드 추가(신규 플랜부터, 없으면 프론트 기본값)
- **DB: meals.completed_at 추가 (리비전 0006) → GATE 3**
- 게스트: 완료 버튼·레시피 시트 클릭 = 기존 가입 게이트/예산 우선 규칙 유지 (샘플 매트릭스 레시피는 간단 기본 조리법 표시)

### Out of Scope (이관)
- 완료 → 가상 냉장고 자동 차감(팀원 fridge 조율), 완료 기반 예산 무드 실계산(아낀 식비 등), 주 단위 주문 생성(order 기획)

## 영향 영역
frontend(끼니 행·바텀시트) / backend(완료 API·MealOut 확장·LLM 옵션 — mealplan 도메인, 팀원 리뷰) / **DB: meals.completed_at → GATE 3** / 마트 연동 없음

## 기능 요구사항
| ID | 요구사항 | 우선순위 |
|----|----------|---------|
| FR-501 | 완료 설정/해제 API: PUT /mealplans/{planId}/meals/{mealId}/completion {completed} — 본인 스코프(CWE-639) | P0 |
| FR-502 | MealOut 확장: steps(단계 배열로 파싱), completedAt, timeMinutes?/difficulty? (하위 호환 옵셔널) | P0 |
| FR-503 | 끼니 행 완료 버튼 — 프로토타입 스타일, 낙관적 갱신+실패 롤백, 게스트는 게이트 | P0 |
| FR-504 | 레시피 바텀시트 — 프로토타입 1:1(메타 3칩·재료 칩·단계 리스트·닫기), 끼니 행 클릭으로 오픈 | P0 |
| FR-505 | LLM 생성 스키마에 time_minutes·difficulty(쉬움/보통/어려움) 옵션 추가 — 신규 플랜부터, 부재 시 프론트 기본값("약 20분"/"쉬움") | P1 |
| FR-506 | 완료 상태는 냉장고 차감·예산 실계산의 전제 데이터 — fridge/order 기획에 접점 명시 | P0 |

## 데이터 요구사항 (GATE 3)
- `meals.completed_at timestamptz NULL` (리비전 0006, down_revision=0005) — 완료=시각, 해제=NULL

## 보안 (CWE)
CWE-639 플랜·끼니 본인 스코프 / 입력은 completed boolean 뿐

## 글로벌 고려
난이도·기본 시간 라벨 i18n, steps 는 생성 로캘 그대로 표시
