# 진입 순서 재정렬 + 가입 온보딩(3단계) 기획서

## 개요 | 배경 및 목적

`/` 진입 순서를 바로잡고, 가입 직후 온보딩을 **Claude Design 프로토타입의 3단계와 동일하게** 구현한다.
(프로토타입 원본: 디자인 프로젝트 `지란고비.dc.html` onboardStep — 스크린샷 3장으로 사용자 확정, 2026-07-09)

## 확정 온보딩 흐름 (프로토타입 1:1)

```
가입 완료 → /onboarding
STEP 1/3  가구 구성원을 설정하세요
  - 가구 유형 빠른 선택 (1인~5인 프리셋 버튼)
  - 구성원 카드 리스트: 유형 배지(성인 남/성인 여/어린이/유아 등) + 나이(세) −/+ 스테퍼 + 삭제(×)
  - 구성원 추가 버튼(유형별), 상단에 "N인 가구" 칩 + "구성원에 맞춰 영양·식사량을 자동 계산해요"
STEP 2/3  한 달 식비 예산을 잠가요
  - 월 예산 대형 금액 + 슬라이더 (범위는 구성원 수 기반: 1인 최소 ₩80,000 · 권장 ₩130,000/인 — 예: 5인 최소 ₩400,000, 권장 ₩650,000)
  - "N인 · 1인 ₩xx · 권장 ₩xx" 칩, 예산 수준 피드백 배너(알뜰/적정/여유)
  - 예산 락 토글 (활성 시 예산 초과 자동 차단 문구)
STEP 3/3  어떤 식단을 원하세요?
  - 선호하는 음식 종류 · 복수 선택: 한식/양식/일식/중식/분식/샐러드·채식 (아이콘 카드)
  - 식단 방향 · 하나 선택: 건강·영양/다이어트/든든·푸짐/아이 입맛 (기존 enum 매핑)
  - CTA "이 조건으로 식단 짜기" → 저장 + 즉시 식단 생성(mealplan) → 생성 로딩 → 홈(식단 표시)
```
- 각 스텝 [이전/다음], 게스트 예산안 이전 성공 가입자는 STEP 2 에 이전값 프리필
- 정확한 비주얼·문구·수치는 디자인 파일 마크업에서 추출 (설계/구현 단계)

## 진입 순서 (사용자 확정)
- 유효 세션 회원 → **바로 회원 홈**. 단 식단 없으면 빈 히어로 전면 금지 — **샘플 채워진 홈 + "내 식단 만들기" 배너**
- 게스트 → 게스트 홈. **로그아웃 이력 재방문자 → [로그인하기/구경하기] 알림** (구경하기=게스트 계속). 신규는 기존 10초 프롬프트
- 온보딩 미완료 회원 로그인 복귀 → `/onboarding` (실화면)

## 범위
### In Scope
진입 순서 3규칙 / 온보딩 3스텝(위) / household 저장(구성원 유형+나이) / budget_plans 확장(예산 락 여부·선호 음식) / 완료 즉시 식단 생성 연결 / 로그아웃 이력 마커
### Out of Scope
온보딩 재진입·수정 화면(설정 기획), 구성원별 영양 개인화 로직 자체(저장·LLM 전달까지만), fridge/order/store

## 영향 영역
- [x] frontend — 온보딩 3스텝 화면(프로토타입 1:1), 진입 분기·알림, 홈 배너
- [x] backend — household 저장/조회 API, budget_plans 확장 반영, (기존 mealplan 생성 연결)
- [x] DB — **household_members 신규 + budget_plans 확장(locked, cuisines) → GATE 3**
- [ ] 마트 연동 — 없음

## 관련 도메인
household(중심) · budget(예산 락·선호) · mealplan(완료 시 생성) · auth(진입 분기)

## 필요 API (초안 — 설계 확정)
- `PUT /api/v1/households/me` — 구성원 배열 [{type, age}] 저장 (최대 10명, 서버 재검증)
- `GET /api/v1/households/me`
- `POST /budget/plans` 확장 또는 `PUT` — locked·cuisines 포함, 기존 플랜 갱신 허용(온보딩)
- 기존: `POST /mealplans` (완료 시 호출, preferences=선호 음식)

## 데이터 요구사항 (GATE 3)
- **`household_members` 신규**: user_id FK, member_type enum(infant/child/teen/adult — 성별 구분은 표시용 라벨로, 저장은 설계 판단), age smallint, position — 서버 재검증(나이-유형 범위 정합)
- **`budget_plans` 확장**: `locked boolean`, `cuisines`(선호 음식 배열) — 마이그레이션 필요
- budget_plans.household_size 는 구성원 수와 동기화

## 보안 (CWE)
