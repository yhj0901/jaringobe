# JARINGOBE (자린고비) 사업계획서 요약

> 원본: 해커톤 사업계획서 (`지란고비 프로젝트.zip` — JARINGOBI 사업계획서 / 패밀리 밥상 제로 프로토타입).
> 모든 에이전트는 기획/설계/구현 시 이 문서를 제품 방향의 기준으로 삼는다.

## 한 줄 정의

**예산 안에서 식단을 자동으로 짜고, 가상 냉장고로 버려지는 식재료를 0으로 만들고, 식재료를 자동으로 주문하는 알뜰 식생활 플랫폼.**
슬로건: *GO BE SMART, SAVE BIG. — 버리지 말고, 똑똑하게.*

## 타깃 사용자

- 1인 가구부터 5인 대가족까지 — 고물가 시대에 매 끼 메뉴 고민과 장보기에 지친 모든 가정
- 대표 페르소나: 성인 2명 + 성장기 자녀 3명(9·7·6세)의 5인 가족

## 핵심 문제

1. 매번 세 끼 메뉴를 고민하는 피로감 → 배달·외식 지출 증가
2. 영양가 높은 식사보다 편리한 인스턴트 식품 의존
3. 남은 식재료 폐기 + 식비가 어디로 새는지 파악 불가

## 목표

한 달 식비에 **예산 락(Budget Lock)** 을 걸고, 영양 있는 집밥으로 가족 건강과 비용을 동시에 잡는다.

## 3가지 핵심 기능

| # | 기능 | 내용 |
|---|------|------|
| 1 | **예산 내 식단 자동 생성** | 가구 구성·식단 방향(건강/다이어트/든든/아이 입맛)·선호 음식에 맞춰 연동 마트의 재료를 검색해 예산 락 안에서 한 달 식단 자동 생성 (AI 식단 + 조리법) |
| 2 | **가상 냉장고 — 버리는 재료 0** | 배송 시 재료 자동 등록, 식사 완료 체크 시 자동 차감(Zero-UX), 유통기한 임박 재료는 다음 식단에 우선 배치 |
| 3 | **식재료 자동 주문** | 연동 마트 계정으로 주 1~2회 예산 내 자동 결제·배송. 냉장고에 남은 재료만큼 다음 주문량 자동 감산(동적 감산) |

## 선순환 루프 (제품의 핵심 동작)

```
① 입력 (가족·예산 락 설정)
  → ② AI 식단 + 자동 주문 (최저가 검색 → 식단 → 자동 결제)
    → ③ 가상 냉장고 (식사 완료 시 자동 차감)
      → ④ 동적 감산 (남은 재료만큼 다음 주 장보기 비용 자동 차감 = 예산 방어)
        ↩ ②로 순환
```

## 프로토타입에서 검증된 기능 목록

- 소셜 로그인 (카카오 · 구글 · 애플)
- 가구 구성원 설정 (1인~다인 · 유아/어린이/청소년/성인)
- 예산 락 + 예산 무드 UI (남은 예산, 절약 금액, 폐기 절감 지표)
- AI 식단 + 조리법 (일별 아침/점심/저녁, 식사 완료/취소)
- 가상 냉장고 자동 차감 (재고·수량·유통기한, 임박 배너)
- 자동 주문 (주 1회/2회 선택, 다음 결제일, 장바구니 추천, 동적 감산)
- 스토어 연동 (마켓컬리 · 쿠팡 · SSG · 네이버 / Walmart · Instacart)
- 한국어 · 영어 지원

## 시장 · 글로벌 전략

| 시장 | 전략 |
|------|------|
| 🇰🇷 국내 | 공식 자동 결제 API 부재 → 사용자가 보유한 마트 계정을 연동해 장바구니·주문 자동화 (마켓컬리/쿠팡/SSG/네이버) |
| 🇺🇸 글로벌 (미국) | 잦은 배송의 숨은 비용(수수료·팁) 방어 → **주 1회 대량 배송 + 식재료 수명 극대화 알고리즘**. Walmart · Instacart Connect API 연동 추진 (조사 중) |

→ **개발 시사점**: 처음부터 다국어(ko/en), 다중 통화(KRW/USD), 타임존, 국가별 마트 어댑터를 고려한 구조로 설계해야 한다.
