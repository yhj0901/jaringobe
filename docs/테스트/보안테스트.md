# 보안 테스트 결과 — 게스트 홈 + 소셜 로그인

> 2026-07-09 / 기준: docs/설계/security-design.md CWE 대응표. 방법: pytest(단위·통합) + 로컬 E2E(curl)

| # | CWE | 케이스 | 방법 | 결과 |
|---|-----|--------|------|------|
| S-01 | CWE-352 | 변조/만료 state 콜백 거부 | E2E TS-07 + pytest | **PASS** — 302 `/login?error=AUTH_INVALID_STATE` |
| S-02 | CWE-601 | `next=https://evil.com` open redirect | E2E TS-03 + pytest | **PASS** — 인가 URL 에 evil.com 미포함, 서버측 상대경로 화이트리스트 |
| S-03 | CWE-287 | 동일 이메일 자동 계정 통합 금지 | pytest | **PASS** — 별도 계정 생성 + notice 쿼리만 |
| S-04 | CWE-384/613 | refresh 회전·재사용 감지 | pytest | **PASS** — 재사용 시 유저 전 세션 폐기 + 401 AUTH_TOKEN_REVOKED |
| S-05 | CWE-522 | refresh 원문 저장 금지 | pytest + 스키마 | **PASS** — SHA-256 해시(char 64)만 |
| S-06 | - | 미인증 접근 401 | E2E TS-01/11 | **PASS** — `{"detail":{"code":"AUTH_REQUIRED"}}` 규격 일치 |
| S-07 | CSRF | Origin 불일치 403 | E2E TS-05 | **PASS** — FORBIDDEN_ORIGIN (헤더 부재 시 통과는 설계 재량 — SameSite=Lax 1차 방어) |
| S-08 | CWE-307 | /auth/* rate limit | E2E TS-06 | **PASS** — IP 버킷 10회/분 초과 시 429 (엔드포인트 합산) |
| S-09 | CWE-200 | 오류 응답 스택 트레이스 미노출 | E2E TS-07 | **PASS** |
| S-10 | CWE-20/602 | budget/plans 서버 전량 재검증 | pytest | **PASS** — 인원/금액 범위/통화/방향 각각 422 |

**추가 확인**: 시크릿 하드코딩 없음(.env 전용, 커밋 검사), provider access token 저장 안 함, 게스트 localStorage 에 PII/토큰 없음(키 화이트리스트 테스트), 쿠키 HttpOnly/SameSite=Lax/refresh Path 한정(pytest).

**주의 항목 (버그 아님)**
- `COOKIE_SECURE=false` 가 로컬 기본값 — **배포 시 true 필수** (.env.example 주석 명시됨)
- rate limit 인메모리 — 멀티 인스턴스 배포 시 Redis 교체 (architecture.md 확장점 기록됨)

**결론: 보안 테스트 10/10 PASS, 취약점 발견 0건**
